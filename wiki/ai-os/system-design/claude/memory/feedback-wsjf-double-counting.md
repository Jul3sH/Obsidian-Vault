Apply the Value vs RO counterfactual test BEFORE proposing a Value score — do not just display the rule and then skip the test when scoring.

**Why:** In a /project-planner session (17 Jun 2026), the double-counting criteria were displayed correctly but then Value was scored at 5 by conflating "what the repositioned profile enables for the job search" with the intrinsic payoff of the project itself. The user caught this. The correct test is: "if someone handed you the outcome without doing the Project, would those downstream opportunities still be blocked?" — if the answer is no, those effects belong in RO, not Value. Phase 1 scored Value=5 when the correct score was Value=1; the downstream job-search effects belong entirely in RO.

**How to apply:** Before proposing any Value score, explicitly run the counterfactual test out loud: "The intrinsic payoff of THIS project alone, if the outcome were handed over without doing the work, is X." Only count effects that would be lost even if the outcome were delivered for free. Everything else is RO.
