# Build the system, then guarantee it gets used

**The pattern:** Julian gets the dopamine reward from *designing* a system (novel, creative, open-ended), not from *running* it (repetitive, boring, executive-function-heavy). So he builds systems - the Jira/sprint system being the canonical case - then abandons them the moment they stop being novel. This is distinct from dropping a *working* system for a shiny one; here the system is never adopted at all. It looks like productivity (something got built) but produces nothing.

**Why it matters:** Building without adopting is wasted effort, and the abandonment is *silent* - no signal fires, so it repeats every time he builds something. This is an ADHD trait (build = reward, run = discipline gap), not a one-off.

**How to apply:**
- When Julian starts building any new system / tool / process / automation - or has just built one - ask: **"What's the adoption forcing-function, and is this going in the systems register?"**
- A system isn't "done" when designed; it's done when it's been *used* for ~2 weeks and survived the boredom. Push back on treating a built-but-unused system as complete.
- Require an **adoption forcing-function named at creation**: a scheduled first-use, a trigger that starts it, or an accountability surface that flags non-use. No forcing-function = predicts abandonment.
- Add or update a row in `wiki/performance/systems-register.md` (the Status + Adoption forcing-function columns are the accountability lever).
- Watch for the sibling failure - the **false prerequisite** (self-sabotage at the starting gate: "I can't use it until I've built the perfect X"). Test it with the crude version immediately. See `wiki/performance/the-human-mind/self-sabotage-and-fear.md`.
- **Distinct from `feedback-overanalysis-check`:** that one is about over-*building* detail into work that doesn't need it; this one is about not-*adopting* what's already built.

Full context: `wiki/performance/working-with-yourself/adhd-aware-work-patterns.md` (§ Build vs Adopt) and `wiki/performance/systems-register.md`.
