# Apollo Person Match

Resolves a list of existing contacts (name + email) to LinkedIn URLs and enriched
profile fields using the Apollo `people/match` endpoint. This is the first step
in the LinkedIn enrichment pipeline — it bridges your contact list to the LinkedIn
enrichment skill that runs next.

## What this skill does

Takes rows from a Google Sheet that already have name and email populated, calls
Apollo for each person, and writes a defined set of fields back to the same rows.
After this skill runs, each matched row has a `linkedin_url` and `linkedin_username`
that the `linkedin-enrichment` skill can use.

## How it fits in the pipeline

```
[your sheet]          →   apollo-person-match   →   linkedin-enrichment
(name + email,            (matches each person       (reads rows with
 optional extras)          via Apollo API,            linkedin_username,
                           writes LinkedIn URL        enriches and writes
                           and profile fields back)   summaries back)
```

Run this skill first. Run `linkedin-enrichment` second.

## Prerequisites

```bash
pip install gspread
```

Two environment variables required:
- `APOLLO_API_KEY` — from https://app.apollo.io/#/settings/integrations/api
- `GOOGLE_SERVICE_ACCOUNT_JSON` — path to your Google service account key file
  (or use OAuth — see `references/gcloud-setup.md`)

## Fields written back to the sheet

Apollo fields overwrite existing sheet values. If Apollo returns null for a field,
the existing cell value is preserved.

| Sheet column | Source | Notes |
|---|---|---|
| `first_name` | Apollo `first_name` | Always written if returned |
| `last_name` | Apollo `last_name` | Always written if returned |
| `title` | Apollo `title` | Overwrites existing |
| `email_address` | Apollo `email` | Overwrites existing |
| `linkedin_url` | Apollo `linkedin_url` | Overwrites existing |
| `linkedin_username` | Extracted from `linkedin_url` | Regex — no LLM needed |
| `organization` | Apollo `organization.name` | Overwrites existing |
| `company_website` | Apollo `organization.website_url` | Overwrites existing |
| `corporate_phone` | Apollo `organization.phone` | Overwrites existing |
| `mobile_phone` | Apollo `mobile_phone` | Overwrites existing |
| `apollo_match_status` | Set by this skill | See status values below |

## Status column: `apollo_match_status`

| Value | Meaning |
|---|---|
| `pending` | Not yet processed (or empty — treated the same) |
| `matched` | Apollo found the person and returned a LinkedIn URL |
| `no_linkedin` | Apollo matched but returned no LinkedIn URL |
| `failed` | API error or no match found |

Rows with `no_linkedin` or `failed` will not have a `linkedin_url` and will be
skipped by `linkedin-enrichment`. Use `--mode reset-failed` to retry them.

## Usage

```bash
# Process a batch of 10 contacts
python scripts/apollo_person_match.py \
  --sheet-id "YOUR_SHEET_ID" \
  --worksheet "Sheet1" \
  --batch-size 10 \
  --mode run

# Check pipeline progress
python scripts/apollo_person_match.py \
  --sheet-id "YOUR_SHEET_ID" \
  --mode stats

# Reset failed rows for retry
python scripts/apollo_person_match.py \
  --sheet-id "YOUR_SHEET_ID" \
  --mode reset-failed
```

## Sheet ID

The Sheet ID is in the Google Sheets URL:
`https://docs.google.com/spreadsheets/d/THIS_IS_YOUR_SHEET_ID/edit`

## Rate limits and credits

Apollo `people/match` costs one credit per call. The script sleeps 0.5s between
calls. On a 429 response it sleeps 60s and retries once. For large contact lists,
run multiple times with `--batch-size 10` rather than processing everything at once.
See `references/apollo-match-api.md` for full rate limit details.

## Error handling
- Row missing email → skipped with warning
- Apollo 429 → sleep 60s, retry once
- No match returned → `apollo_match_status = "failed"`
- Match returned, no LinkedIn URL → `apollo_match_status = "no_linkedin"`
- Missing sheet columns → auto-added on first run
