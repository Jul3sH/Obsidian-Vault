# Standup Skill — End-of-Day Review

Runs at 4 PM HKT on weekdays to close out the day. Reads the active Jira
sprint, asks 3 questions, updates story statuses based on your answers, and
logs the review for reference.

**Duration:** 5 minutes max. Three questions. No fluff.

## Step 1 — Fetch Sprint Status

**JQL:**
```
project = BWS AND sprint in openSprints() ORDER BY priority ASC
```

Use `searchJiraIssuesUsingJql`. Extract for each story:
- Key (BWS-N)
- Summary
- Status (To Do / In Progress / Done)
- Story points (`customfield_10016`) = hours

**Display:**
```

SPRINT STATUS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  BWS-2  Integrate Platform & Send    In Progress   8h
  BWS-3  Run Campaign & Measure       To Do        16h
  BWS-4  Build Enrichment Skills      To Do        40h
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Sprint: Sprint 1 — 7–13 May 2026
```

If no active sprint exists: tell the user and offer to run `/sprint-plan`.

## Step 2 — Ask 3 Questions

Ask all three in a single message:

```
Three questions:

1. What got done today?
2. Where did you leave off / what's still in progress?
3. Any blockers, scope creep, or changes needed?
   (Did you work on anything outside the current story's Definition of Done?)
```

Accept freeform text. Don't force structure — parse it yourself.

**On the scope creep probe:** It's embedded in question 3 to avoid adding ceremony overhead.
If the user mentions working on something outside the current story's DoD, surface it:
- Trivial (< 30 min, incidental): note it in the log, no action needed
- Meaningful addition: "That sounds like it belongs in a new story — want me to create one in the backlog now or flag it for sprint planning?"
- Pattern emerging (2nd time this week): "This is the second time scope has expanded — worth naming at Friday's retro."

---

## Step 3 — Interpret and Confirm

Parse the user's answers to identify:
- **Completed** → transition to Done
- **In progress** → transition to In Progress (if not already); capture where they left off
- **Blocked** → add a Jira comment with the blocker; don't change status
- **Not touched** → leave as-is

Summarise your interpretation before writing anything:

```
Got it. Here's what I'll update in Jira:

  BWS-2 → Done ✓
  BWS-3 → In Progress (note: left off at email template setup)
  BWS-4 → no change
  Blockers: none

Confirm? (yes / correct me)
```

Wait for confirmation before touching Jira.

---

## Step 4 — Update Jira

**Completing a story:**
- Fetch available transitions: `getTransitionsForJiraIssue`
- Transition to Done: `transitionJiraIssue`

**Moving to In Progress:**
- Transition if not already there
- If user described where they left off, add a comment via `addCommentToJiraIssue`:
  `"Left off at: [user's note]. Resuming tomorrow."`

**Blocked stories:**
- Add comment: `"Blocker: [user's description]"`
- Leave status unchanged — blocked is a note, not a transition

---

## Step 5 — Log and Close

**Append one row to `wiki/ai-os/logs/standup-log.md`:**

```
| [date] | [completed stories] | [in-progress + where left off] | [blockers or "none"] | [scope creep noted or "none"] |
```

**Final output:**
```
Standup logged. ✓

  Completed:    BWS-2 (8h) → Done
  In Progress:  BWS-3 — left off at email templates
  Blockers:     None

Sprint remaining: 16h (BWS-3) + 40h (BWS-4) = 56h
→ https://agileict.atlassian.net/jira/software/projects/BWS/boards
```

## Edge Cases

**Nothing got done today:**
- Accept it without judgment. Log "no progress" and move on.
- Don't ask why — that's what `/retro` is for on Fridays.

**Flow state after 4 PM / manual run:**
- Runs identically whenever triggered. The log captures actual time.
- Useful for capturing a late session before context is lost.

**Story completed that was still To Do (skipped In Progress):**
- Transition directly to Done. Add a comment noting it was completed same-day.

**Multiple stories completed:**
- Transition all of them in sequence.

**Sprint is empty:**
- Tell user and stop. Offer `/sprint-plan`.
