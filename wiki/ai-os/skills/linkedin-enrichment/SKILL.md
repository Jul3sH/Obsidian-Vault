# LinkedIn Enrichment

Reads contacts from a Google Sheet that already have a `linkedin_username` (populated
by the `apollo-person-match` skill), fetches their LinkedIn profile and recent posts
via the LinkedIn Data API, generates AI summaries of both, and writes the results
back to the same sheet.

## What this skill does

For each pending contact row:
1. Calls the LinkedIn Data API to fetch the full profile
2. Cleans and structures the profile fields
3. Sends the profile to an LLM and generates a cold-email-ready summary
4. Calls the LinkedIn Data API to fetch the person's 3 most recent posts
5. Sends the posts to an LLM and generates a 2-paragraph narrative summary
6. Writes both summaries and status flags back to the sheet

## How it fits in the pipeline

```
[your sheet]          →   apollo-person-match   →   linkedin-enrichment
(name + email,            (resolves LinkedIn         (this skill — reads
 optional extras)          URL and username,          linkedin_username,
                           writes back to sheet)      writes summaries back)
```

Run `apollo-person-match` first. Run this skill second.

## Fields written back to the sheet

| Sheet column | Written by | Values |
|---|---|---|
| `about_linkedin_profile` | Profile enrichment | AI-generated summary (2-3 paragraphs) |
| `profile_summary_scrape` | Profile enrichment | `completed` / `failed` |
| `recent_posts_summary` | Posts enrichment | AI-generated narrative (2 paragraphs) |
| `posts_scrape_status` | Posts enrichment | `scraped` / `failed` |
| `linkedin_username` | If missing, extracted from `linkedin_url` | |
| `extract_username_status` | If username extracted | `finished` |

## Prompts

Both prompts are used verbatim as they appeared in the source workflow.

**Profile summary** — used for `about_linkedin_profile`:
> Summarize the LinkedIn profile data for a lead to cold email. Include relevant
> bits for personalized outreach. Focus on current role, career trajectory,
> expertise, and cold email hooks. Keep under 3 paragraphs.

**Posts summary** — used for `recent_posts_summary`:
> Summarize the most recent posts collectively in no more than two short paragraphs.
> Focus on main themes, tone, and recurring interests. Synthesize into a narrative —
> avoid listing posts separately.

## Usage

```bash
# Enrich both profiles and posts (most common)
python scripts/run_enrichment.py \
  --sheet-id "YOUR_SHEET_ID" \
  --worksheet "Sheet1" \
  --batch-size 2 \
  --provider anthropic \
  --mode both

# Profile summaries only
python scripts/run_enrichment.py --sheet-id "YOUR_SHEET_ID" --mode profile

# Posts summaries only
python scripts/run_enrichment.py --sheet-id "YOUR_SHEET_ID" --mode posts

# Check pipeline progress
python scripts/run_enrichment.py --sheet-id "YOUR_SHEET_ID" --mode stats

# Reset failed rows for retry
python scripts/run_enrichment.py --sheet-id "YOUR_SHEET_ID" --mode reset-failed
```

## Batch size

Keep `--batch-size` low (2-5). Each row makes two RapidAPI calls (profile + posts)
and two LLM calls. Running in small batches respects API rate limits and means
partial progress is saved to the sheet even if a run is interrupted.

## LLM provider

Default is Claude Haiku (`anthropic`) — cheaper than GPT-3.5 with equivalent
summarization quality. See `references/llm-providers.md` for a full cost and
quality comparison including Groq and Ollama options.

## Error handling
- Row missing `linkedin_username` → skipped with warning (run apollo-person-match first)
- RapidAPI 429 → sleep 30s, retry once
- Empty posts array → `posts_scrape_status = "failed"` (some users don't post)
- LLM timeout → status set to `failed`; use `--mode reset-failed` to retry
- Missing sheet columns → auto-added on first run
