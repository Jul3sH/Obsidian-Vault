## Phase 1 — Define the Epic

### Step 1.1 — Workstream

**Claude:** Great — let's structure this properly. First question: which workstream does this fall under — career, performance, finance, personal, relationships, technical, or wellbeing?

This sounds like it could sit in either **performance** (if the goal is personal effectiveness and learning) or **technical** (if it's about building a reference library for AI tools). My instinct is **performance** — specifically the "working-with-yourself" or "working-with-others" angle of becoming an AI-native professional. Does that feel right, or is this more about building technical knowledge?

**User:** Yeah, performance makes sense. It's about getting better, learning things. Though honestly it's also technical because I'm building skills and tools...

**Claude:** I'd keep it under **performance** — you're building capability in yourself, not maintaining a reference library. The technical wiki is for "how things work"; this Epic is about what you can do differently as a result. We can cross-link to technical when we write the files.

So: **workstream = performance**. Now the important one...

---

### Step 1.2 — The Objective

**Claude:** In one sentence, what outcome are you trying to achieve? Not what you'll do — what will be *different or better* when this is done?

**User:** I want to learn more about AI tools and get better at using them. Like, build some projects, maybe take a course...

**Claude:** That's activity-focused — it tells me what you'll *do*, not what changes as a result. Let me push on this: imagine it's three months from now and this Epic is complete. What's true about your professional life that isn't true today?

Is it: you can build working AI automations independently? You're competitive for AI-adjacent roles? You have a portfolio that demonstrates capability? Something else?

**User:** I think... I want to actually be able to build AI tools end-to-end without getting stuck. And I want it to count for something career-wise — like, if someone asks "what have you done with AI?" I have a real answer.

**Claude:** Good. Let me rephrase that precisely: *"Be a credibly capable AI builder — able to ship working end-to-end tools independently and articulate that capability for career purposes."* Does that capture it? Or do you want to add a time horizon — e.g. "within 90 days"?

**User:** Yes, that's it. Let's say 90 days.

**Claude:** Objective locked: *"Within 90 days, become a credibly capable AI builder — able to ship working end-to-end tools independently and articulate that capability for career and professional positioning."*

Back to KRs. "Get better at using Claude" isn't measurable — I can't tell when you're done. Let me challenge you: what would *prove* you're better? Is it that you've shipped something that uses the Claude API? That you've built a skill or automation that runs in production? Give me a concrete artefact or threshold.

**User:** OK fair. Maybe... I've built and shipped 3 working AI tools or automations end-to-end. That's concrete.

**Claude:** That's a solid KR. Let's lock it in with a time box: *"Ship 3 working AI tools or automations end-to-end by July 31, 2026."*

**User:** The career positioning piece. Like being able to talk about it. Maybe publish something publicly about what I've built?

**Claude:** Good. Let's make it measurable: *"Publish 3 write-ups (LinkedIn posts or wiki articles shared publicly) about AI tools or projects built, by July 31, 2026."* Binary, time-boxed, directly serves the "articulate the capability" part of the Objective. KR2 confirmed?

**User:** Yes, that works.
