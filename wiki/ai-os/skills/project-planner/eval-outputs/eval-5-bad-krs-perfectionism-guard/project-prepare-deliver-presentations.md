# Project: Prepare and Deliver 3 Presentations; Gather Credibility Feedback

## What this delivers
**Key Result:** Deliver 3 presentations to leadership/all-hands; gather peer feedback showing 80%+ "credible" rating by August 31

## Definition of Done
- Prepare 3 presentations (targeting leadership or all-hands audiences)
- Each presentation uses clear storytelling (arc: situation → problem → solution)
- Each uses clean, uncluttered visuals
- Deliver all 3 presentations by August 31
- After each presentation, send a brief 1-question survey to 5–10 attendees: "How credible and confident did I come across?" (scale 1–5, or yes/no)
- Compile feedback showing net 80%+ "credible" or "agree" rating
- Document any patterns in feedback for learning

## Effort Estimate
**Focused time:** 8–12 hours (over 3 months, May–August)  
**Calendar time:** 12+ weeks (depending on presentation scheduling)

## Execution Plan

### Phase 1: Identify Presentations (May)
- What are 3 topics you can present on? (Product strategy, team achievements, cross-functional initiative, technical insight, culture topic?)
- Who's the audience? (Leadership? All-hands? Cross-functional team?)
- When should they happen? (Aim for June, July, August)

### Phase 2: Prepare (June–July)
For each presentation:
1. **Outline:** 5-minute write — what's the story? (Situation → Challenge → Your Action → Result/Insight)
2. **Visuals:** 5–8 slides max. One idea per slide. No text-heavy bullets.
3. **Narrative:** Script the opening (30 sec) and closing (20 sec) so you start and end strong.
4. **Practice:** Run through once with a peer. Get feedback on pace and clarity.

Estimated time per presentation: 2–3 hours.

### Phase 3: Deliver (June–August)
- Schedule presentations by June 15
- Deliver by August 31
- Each presentation: 20–30 minutes + Q&A

### Phase 4: Gather Feedback (after each, by August 31)
- After each presentation, send a 1-question survey (Google Form, Slack poll, etc.)
- Question: "How credible and confident did I come across?" (1–5 scale or Yes/No)
- Target 5–10 responses per presentation
- Goal: 80%+ rating "credible" or "agree"

## Feedback Success Criteria
| Rating | Counts As | Target |
|--------|-----------|--------|
| 4–5 or "Agree" | Credible | 80%+ |
| 2–3 | Neutral | OK but watch |
| 1 or "Disagree" | Not credible | Flag for retool |

## Notes
This is the largest project (M) but spread over 3 months, so effort is manageable. The key is starting early so you have time to prep and practice.

**Tip:** Credibility in presentations comes from:
- Knowing your material (so you can speak naturally, not read)
- Clear structure (audience can follow)
- Authentic presence (not performing; being real)
- Storytelling (not just data dump)

This project is about practicing and demonstrating all four.
