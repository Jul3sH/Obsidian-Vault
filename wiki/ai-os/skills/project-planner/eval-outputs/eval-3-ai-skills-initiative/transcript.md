## Eval Notes

**Skill behaviours observed:**
- Correctly pushed back on activity-focused objective and guided reframe to outcome-focused statement
- Enforced time boxes on all 3 KRs without prompting from user
- Challenged initial M t-shirt estimate and got correction to L
- Used counter-factual questioning for WSJF (asked "what gets worse" not "score this")
- Proposed scores and asked for confirmation rather than making user self-score
- Flagged the WSJF/dependency tension on course completion and resolved it with a concurrent-running note
- Stayed efficient — no unnecessary digressions
- Correctly identified first project explicitly at close

**Potential issues to flag for eval:**
- The WSJF ranking puts the course (a logical dependency) last — the skill resolved this with a note, but the ranking table alone could mislead an ADHD user into deferring the course entirely. Consider whether the epic template should add a "sequencing constraints" section.
- XS job size for enrolment maps to 1 in the formula, which dramatically inflates the WSJF (8.0 vs next-best 3.5). This is mathematically correct but could feel surprising. The skill did not flag this — could be worth noting.
