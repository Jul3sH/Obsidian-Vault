# project-planner — Testing Protocol

## Eval Suite

Test cases are defined in `evals/evals.json`. Each eval simulates a full `/project-planner` session, including Phase 3 (wiki write). This means **eval agents create real files in the wiki** — they are indistinguishable from production projects unless cleaned up.

Run evals via the `/skill-creator` flow. Each eval spawns two subagents: one with the skill, one baseline (no skill).

---

## ⚠️ Post-Eval Cleanup — Human Confirmation Required

Eval agents write real Project files to `wiki/projects/` and update `wiki/projects/_index.md`. These must be removed after each eval run.

**Do NOT delete eval-created files without human confirmation.** The cleanup procedure is:

### Step 1 — Identify eval-created files
After running evals, check `wiki/ai-os/logs/log-2026-Q2.md` (or current quarter log) for entries dated on the eval run date with type `Setup` referencing `wiki/projects/`. Also list the files:

```bash
ls wiki/projects/
```

Cross-reference against known real Projects. Any Project file created on the eval run date that the user did not define in a real session is an eval artifact.

### Step 2 — Present the list to the user

Show the user the list of files to be deleted:

> "The following Project files were created by eval agents during today's eval run. These are not real Projects. Can I delete them and remove them from the index?"
>
> - `wiki/projects/[file1].md`
> - `wiki/projects/[file2].md`
> - (etc.)

**Wait for explicit user confirmation before proceeding.**

### Step 3 — Remove after confirmation
Once the user confirms:
1. Delete each eval-created `.md` file from `wiki/projects/`
2. Remove the corresponding rows from `wiki/projects/_index.md`
3. Log the cleanup in the operations log:
   `| [date] | Maintenance | Removed [N] eval-generated Project files from wiki/projects/ after user confirmation. |`
