# Top-Down Execution — Start With the Deliverable

## The Pattern to Break

A recurring failure mode: starting from the bottom up, going into deep research, learning tangentially interesting things, and never producing the actual output. Then panicking when time runs out and suddenly working top-down efficiently under pressure.

The chemicals released under deadline urgency are what produce focus. The lesson is to manufacture that urgency and top-down discipline from the start.

## Start With the Deliverable

- Open the document, slide, design, or output file immediately and start working in it
- Starting in the deliverable provides clarity about what is actually needed — far more than research does
- Indecision dissolves once you're inside the work
- The direction only becomes clear through doing, not through planning

## The Top-Down Approach

1. Grasp the overall objective first — use SQ3R or similar; don't start grouping tasks before understanding the goal
2. Brainstorm and review options before producing anything detailed
3. Produce a rough top-level version (gut, not looked up)
4. Iterate — improve from a working draft rather than starting from perfect

Doing it from the top down with "good enough" speed is dramatically more effective than bottom-up perfectionism.

## Minimal Investigation — Get Started

- Do minimal research, then get started; research the actual roadblocks as they appear
- The roadblocks you anticipated usually aren't the real ones
- Watching/reading about how to do something is nowhere near as effective as doing it
- Mental barriers built up through over-research are usually false

## Police Rabbit Holes

- Rabbit holes are learning disguised as work — they feel productive and are usually not
- Set a strict timebox for any research activity before beginning
- Learning must have a constrained scope: what do I need to know to produce this output?
- "Don't unnecessarily deep dive" — if it's not load-bearing for the deliverable, skip it

## Top-Down MVPs

- Produce a minimum viable version first — then refine
- A rough working draft is infinitely more useful than extensive notes toward a perfect one
- MVPs expose the real gaps quickly and allow iteration based on actual feedback

## Read With Purpose

- When reading, know what you're looking for before you start
- If stuck on a concept, read on — the explanation often appears later
- Deep-diving immediately into an unfamiliar concept wastes time; trust the process and continue

## Develop for Here and Now

- Do not build for possible eventualities that may never occur
- Wait until you actually need the knowledge or capability before acquiring it
- Example: spent time preparing for a technology that was never used in the role

## My Pattern

The bottom-up drift pattern is deeply familiar: starting with research, going deep on interesting tangential material, and only switching to top-down execution under genuine deadline pressure — at which point the work gets done quickly. The chemicals released under deadline urgency genuinely do produce the best performance; the task is to manufacture those conditions rather than waiting for them.

## Key Takeaways

- Open the deliverable immediately and start working in it — this creates more clarity than any amount of research
- Top-down, gut-first, iterate is faster and better than bottom-up perfectionism
- Brainstorm and validate options before producing detailed work on a single option
- Rabbit holes feel like progress; they are usually avoidance — timebox all research ruthlessly
- Develop for now, not for eventualities — wait until you need it
