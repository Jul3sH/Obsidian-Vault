# Habits Register

All active habits across all workstreams, with their goal alignment. Habits are recurring behaviours that compound toward goals — distinct from Projects, which are one-off deliverables.

**Goals they support:** [[goals-register]]
**Workstream files** are authoritative for context and principles behind each habit.

---

## How to read this table

- **ID** — short reference (used to populate the Habits column in [[goals-register]])
- **Workstream** — which workstream this habit primarily belongs to
- **Habit** — the specific recurring behaviour (concrete and observable)
- **Frequency** — daily / weekly / Nx per week / etc.
- **Supports Goals** — IDs from [[goals-register]] that this habit compounds toward
- **Status** — active / building (not yet automatic) / paused / retired

## How to add a habit

A good habit entry is:
- **Concrete** — describes a specific observable behaviour, not an intention ("30 min gym session" not "exercise more")
- **Frequency-bound** — has a clear cadence
- **Goal-linked** — references at least one goal ID from [[goals-register]]

When adding a habit:
1. Add a row to the table above
2. Update the **Habits** column in [[goals-register]] for each supported goal ID
3. Update the **Habits** column in the relevant workstream's Active Goals table
